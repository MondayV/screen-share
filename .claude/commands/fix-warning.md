# 指令：修复“连接码过期或无效”错误

## 目标
确保屏幕共享短码（6位连接码）可正常生成、复制、被对方使用加入房间，不再出现“过期或无效”提示。

## 任务
1. **审查信令服务器房间管理代码**（`src/main/signaling-server.js` 或等效文件）  
   - 检查房间创建时的存储逻辑：是否将 `roomId` 正确写入 Map/Set。  
   - 检查是否有“房间过期”机制：如有，请将过期时间延长至 5 分钟，或完全移除（直到房主断开再清理）。  
   - 确保加入房间时，`roomId` 查询是**字符串严格匹配**，无多余空格、无大小写转换。

2. **审查客户端生成/加入房间的逻辑**（`src/renderer/src/WebRTC.svelte` 或连接组件）  
   - 生成房间时，确认发送给信令服务器的消息类型正确（如 `type: 'create-room'`），并且服务器正确处理。  
   - 加入房间时，确认发送的消息体包含正确的 `roomId`，且读取自输入框（可能需要 `.trim()`）。  
   - 检查短码显示是否与复制的内容一致（比如前端显示带空格，复制时保留了空格）。

3. **增加调试日志**  
   - 在信令服务器上为房间创建和加入操作添加 `console.log`，输出 `roomId` 和当前房间列表。  
   - 在前端加入失败时，将收到的错误消息输出到控制台，便于排查。

4. **测试验证**  
   - 使用 `test-two.bat` 启动双实例，主实例点击“开始共享”获取短码。  
   - 在测试实例输入短码并加入，观察控制台日志，确保连接成功。  
   - 如果仍失败，根据日志进一步分析。

## 验收
- 主实例生成短码后，测试实例在 2 分钟内使用同一短码可成功加入。  
- 不再出现“连接码过期或无效”错误。  
- 信令控制台打印清晰的状态信息。
