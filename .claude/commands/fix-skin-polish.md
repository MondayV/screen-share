# 指令：修复皮肤样式三问题（深色覆盖、手账对比度、像素字号）

## 目标
1. 彻底解决“切换皮肤后仍显示深色模式”的样式污染问题。  
2. 提升手账涂鸦皮肤的文字对比度，确保清晰易读。  
3. 将像素比特皮肤的基础字号调整为可读大小，同时保持像素风格。

## 规则
- 仅修改样式文件，不破坏现有功能和布局。
- 所有中文文案保持不变。
- 修复后五款皮肤视觉差异明显，各自独立无交叉污染。

---

## 问题一：皮肤切换后仍为深色模式

### 原因排查
- 可能存在的全局深色样式（如 `body { background: #1e1e1e; color: #fff; }`）未被 `[data-theme]` 作用域覆盖，或优先级高于主题样式。
- 可能皮肤 CSS 文件中变量定义不全，导致部分组件回退到深色默认值。
- 可能 `<html>` 上的 `data-theme` 属性未正确切换或未被 CSS 选择器捕获。

### 修复要求
1. **审查所有全局样式文件**（`global.css`、`app.css` 等），删除任何**无前缀的强制深色声明**（如 `body` 的背景、文字颜色），全部替换为主题变量，且变量应在 `[data-theme]` 内定义。
2. 在每个皮肤 CSS 文件（如 `default.css`、`dark.css`、`cyberpunk.css` 等）中**显式定义所有用到的 CSS 变量**，不允许依赖浏览器默认值。特别注意 `--bg-primary`、`--text-primary` 等核心变量。
3. 确保皮肤 CSS 文件**按正确顺序引入**，且 `[data-theme]` 选择器的特异性足够覆盖全局样式。必要时为皮肤添加 `!important` 仅作为最后手段，优先提高特异性（如 `html[data-theme="doodle"] body`）。
4. **验证**：切换至每款皮肤后，打开 DevTools 检查 `body` 的背景色和文字色，确保完全符合设计，无深色泄露。

---

## 问题二：手账涂鸦文字对比度太低

### 当前问题
米白背景上文字颜色过浅，导致看不清。

### 修复要求
1. 修改 `doodle.css`，将主文字色 `--text-primary` 设为 **深灰或暖黑**，如 `#3A3A3A` 或 `#2C2416`（温暖墨色）。  
2. 辅助文字 `--text-secondary` 至少为 `#6B6B6B`，保证对比度 ≥ 4.5:1（WCAG AA 标准）。  
3. 检查所有文字元素（标题、正文、按钮文字、输入框文字）是否继承正确变量，不出现浅灰色文字。  
4. 若使用了浅色手写字体，可适当加粗（`font-weight: 500` 或 `600`）增强可读性。  
5. 确保选中文本背景与文字对比明显（如选中背景珊瑚粉，文字白色）。

---

## 问题三：像素比特文字太小，看不清

### 当前问题
`Press Start 2P` 或 `Silkscreen` 字体默认 8px 太小，阅读困难。

### 修复要求
1. 在 `pixel.css` 中，将 **基础字号提升至 12px 或 14px**（`--font-size-base: 14px`），所有文本尺寸基于此变量等比放大。  
2. 按钮、标题、输入框等可在此基础上适当加大（如标题 18px，按钮 14px），但保持像素整体比例。  
3. 同时调整 `line-height` 为 1.5 或更高，避免拥挤。  
4. 确保按钮、输入框的内边距相应增大，防止文字过大溢出。  
5. 若需保留像素感，可使用 `image-rendering: pixelated` 加上稍大字号，视觉上仍是像素风格。  
6. 验证：在 1366x768 分辨率下，所有像素文字清晰可读，不模糊不费眼。

---

## 验收标准
1. 皮肤切换瞬间，深色污染消失。默认皮肤亮白，深色皮肤暗灰，赛博朋克暗紫霓虹，手账米白，像素深蓝。  
2. 手账涂鸦下，任何文字在米白背景下都清晰可辨，无视觉疲劳。  
3. 像素比特下，所有文字大小如同普通桌面应用，无需凑近屏幕辨认。  
4. 所有皮肤的功能控件保持完整交互，无错位或样式丢失。
