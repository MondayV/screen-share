# 指令：优化 screen-share 应用，使其贴近专业会议软件

## 你的角色
你是一位资深前端/桌面应用工程师，精通 Electron、Svelte、WebRTC 技术栈。

## 项目仓库
当前工作目录为 screen-share 项目根目录（包含 package.json）。

## 任务

### 第一阶段（核心功能）
1. **互动批注与白板**：创建可协作的批注层（Canvas + DataChannel 同步），实现画笔、文字、橡皮擦工具。
2. **悬浮窗模式**：启动共享后，主窗口最小化，弹出悬浮窗（带说话者视频小窗）。
3. **人像画中画**：共享时在右下角叠加摄像头画面，可拖动调整。
4. **远程控制**：实现共享方对观看方的鼠标键盘控制，需增加安全确认弹窗。

### 第二阶段（增强体验）
5. **会中聊天**：右侧面板聊天窗口，支持文字、表情、文件链接分享（引用[10]）。
6. **音频共享**：共享屏幕时可同步共享系统音频输出，需在 `getDisplayMedia` 中启用 `audio: true`。
7. **绿色角标**：共享时画面四周显示绿色边框提示。

### 第三阶段（完善）
8. **录制与回放**：使用 MediaRecorder 录制远程流并下载。
9. **快捷表情回应**：工具栏增加“举手”、“鼓掌”等快速反应按钮。
10. **UI 统一**：按照上方布局表重构界面，使用专业会议风格，简化按钮数量，底部只保留核心操作。

## 技术要求
- 保持现有全中文界面，所有提示文字用中文。
- 使用 Svelte 组件化开发，每个功能模块对应独立组件（如 `Annotation.svelte`, `Chat.svelte`, `RemoteControl.svelte`, `PictureInPicture.svelte`）。
- 涉及信令的，优先使用现有 WebSocket 连接，不新增通信端口。
- 进行每一步时，先解释改动点、涉及的文件，并给出关键代码片段。
- 完成后提示如何测试（如执行 `npm run dev`）。
