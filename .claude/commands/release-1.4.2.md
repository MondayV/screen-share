# 指令：发布 screen-share v1.4.2

## 目标
基于当前 `main` 分支，完成 v1.4.2 版本的准备工作、提交、打标签并推送至 GitHub，触发发布流程。

## 执行步骤

### 1. 版本号确认与更新
- 检查 `package.json` 中的 `version` 字段是否为 `1.4.2`，若不是则修改为 `1.4.2`。
- 检查 `package-lock.json` 中的版本号是否同步更新（通常修改 `package.json` 后运行 `npm install` 会自动更新，但请确认）。

### 2. 生成 CHANGELOG.md
在项目根目录创建或更新 `CHANGELOG.md`，按以下格式添加 v1.4.2 内容：

```markdown
# 更新日志

## [1.4.2] - 2026-05-14
