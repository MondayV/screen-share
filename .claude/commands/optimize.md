# 优化 screen-share 至腾讯会议体验（分阶段执行）

## 目标
在现有 Electron+Svelte+WebRTC 基础上，实现腾讯会议核心交互：  
- 信令自动连接、画中画、互动批注、悬浮窗、远程控制、聊天、录制

## 规则（严格遵守以节省 Token）
- 仅输出需要修改的文件名及最小 diff，不要完整重写文件。
- 不要解释已存在的代码，除非必须说明。
- 每个阶段独立完成，完成后等待我的“继续”指令再进入下一阶段。
- 所有 UI 文字使用中文。

## 阶段一：信令彻底无感化
1. 主进程内嵌信令服务器（使用 ws 库，端口 3456，监听 0.0.0.0）。
2. 修复 CSP：在 index.html 的 meta 标签添加 connect-src ws://localhost:3456。
3. 移除所有“信令未连接”错误提示，改为自动重连+状态 store。
4. 前端：收到主进程 IPC 信号后才发起 WebSocket 连接，避免抢跑。

## 阶段二：核心会议功能
1. 画中画：共享时摄像头视频浮于右下角，可拖拽。
2. 悬浮窗：开始共享后主窗口最小化，弹出置顶迷你窗口（含发言者视频+快捷控制）。
3. 互动批注：共享画面上叠加 Canvas，画笔/文字/橡皮，通过 DataChannel 同步坐标。
4. 远程控制：通过 DataChannel 传输键鼠事件，主端用 robotjs 执行，需权限确认。

## 阶段三：协作工具
1. 右侧聊天面板：文字+表情，DataChannel 广播。
2. 音频共享：getDisplayMedia 启用 audio: true。
3. 录制：MediaRecorder 录制远程流，结束会议后弹出保存文件。

## 阶段四：UI 精修
1. 布局重构：顶部栏（会议码+头像）、主区域（共享画面）、底部栏（精简按钮）、右侧面板。
2. 绿色角标：共享时画面四周边框。
3. 深色/浅色主题切换。

## 验收
每阶段完成后，我手动测试通过再继续。确保功能可用且无回归。
