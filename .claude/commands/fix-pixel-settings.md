# 指令：修复像素比特皮肤 - 设置界面皮肤主题缺失与下拉框适配

## 目标
在像素比特皮肤下，确保设置面板正常显示“皮肤主题”切换选项，并调整下拉框大小，使其能够完整显示“皮肤主题”四个字，同时保持像素风格。

## 修复要求

### 1. 显示皮肤主题选项
- 检查设置面板组件（如 `SettingsModal.svelte` 或包含皮肤切换的区域），确认其未被像素比特皮肤的 CSS 隐藏或覆盖。
- 如果皮肤切换器是通过条件渲染或依赖全局状态，确保在像素比特皮肤下未被禁用。
- 如果使用了 `display: none` 或 `opacity: 0` 误伤，请修正选择器，确保只作用于装饰元素而非功能组件。

### 2. 下拉框尺寸适配
- 定位到皮肤选择的下拉框（`<select>` 或自定义下拉组件）。
- 在 `pixel.css` 中，为下拉框设置足够宽度，使“皮肤主题”四个字完全可见，例如：
  - `min-width: 160px;`
  - `width: auto;`
  - `padding: 8px 12px;`
- 若使用像素字体 `Press Start 2P`，需增加 `font-size` 至 14px 或 16px 以确保文字可读。
- 调整 `option` 子元素的字号和内边距，保证下拉展开时选项文字也清晰。
- 确保下拉箭头（`appearance: none` 自定义箭头）尺寸匹配文字大小，不遮挡文字。

### 3. 全局一致性
- 修复后，在像素比特皮肤下，设置界面应当完整显示所有选项，皮肤主题下拉框操作自如。
- 切换至其他皮肤时，设置界面仍保持正常，不受影响。

### 4. 测试
- 打开应用，切换到像素比特皮肤。
- 进入设置面板，确认能看到“皮肤主题”标签和下拉框。
- 下拉框能够完整展示当前选中皮肤名称，点击展开选项不重叠、不截断。
- 功能正常，选择皮肤后即时生效。
