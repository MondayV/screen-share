# 指令：操作流程腾讯会议化 + 全自动连接修复（保持界面不变）

## 目标
- 用户打开应用后，无需任何技术配置，**仅通过“开始会议”和“输入短码加入”两步即可完成屏幕共享**。
- 修复所有残留的连接失败问题（CSP、硬编码地址、缺少 TURN、防火墙提示）。
- **保持现有界面布局、皮肤系统、控件样式完全不变**，仅修改交互流程和默认行为。

## 核心原则
- 零手动配置：IP、端口、TURN 全部自动处理。
- 短码即入口：主持方生成 6 位短码，加入方输入即可连接。
- 操作逻辑向腾讯会议看齐：打开应用 → 开始会议（或加入会议） → 共享/观看。

- 文件：`src/renderer/index.html`  
- 修改 `<meta>` 标签的 `connect-src`，允许所有 WebSocket 连接：connect-src 'self' ws://:;

- 文件：所有创建 `new WebSocket` 的位置（如 `src/renderer/src/WebRTC.svelte` 或 `src/lib/ws-manager.js`）
- 替换硬编码的 `ws://localhost:3456` 为：
```js
const signalingUrl = window.location.hostname === 'localhost'
  ? 'ws://localhost:3456'
  : `ws://${window.location.hostname}:3456`;
同时增加重试：连接失败后每 3 秒重试，5 次后提示“无法连接到信令服务，请检查防火墙是否放行端口 3456”。

1.4 防火墙友好提示

文件：主进程信令服务器启动后（src/main/signaling-server.js 或主入口）

在服务器 listening 事件中，通过 IPC 通知渲染进程“信令已启动”，同时检查局域网可达性（简单尝试连接本地 IP），如果检测到可能被防火墙阻止，弹出一次性的系统通知：“请允许 Node.js 通过防火墙，否则局域网内其他设备可能无法连接”。

步骤 2：操作流程极简化（不改界面布局）
2.1 主持方一键开始会议

文件：主持界面组件（如 Host.svelte 或 App.svelte 中的主持逻辑）

现有界面中应有“开始共享”按钮，保留其样式和位置，但修改其行为：

点击后直接开始屏幕共享（如果摄像头已开启则同时发送）。

自动生成 6 位短码（保持现有短码系统）。

短码显示区域保持现有样式，但增加一个“复制短码”图标按钮（轻量，不改变整体布局）。

默认共享配置：

麦克风：默认开启（可点击麦克风按钮关闭）。

摄像头：默认关闭（点击摄像头按钮开启，已实现）。

共享源：默认弹出系统选择窗口（用户可选整个屏幕或窗口），如果已在上一次选择过且勾选“记住”，则自动使用上次选择。

2.2 加入方输入短码即连接

文件：加入界面组件（如 Client.svelte 或 App.svelte 中的加入逻辑）

保留现有输入框和按钮样式，但优化行为：

用户粘贴或输入 6 位短码后，自动去除空格、大写转换。

点击“加入”后，显示“正在连接…”状态（可用现有 loading 组件）。

连接成功后自动全屏显示对方共享画面（现有行为保留）。

连接失败时，给出清晰原因（如“短码无效”“网络不可达”），并提供重试按钮。

2.3 会议中操作简化

工具栏保持现有样式，但调整按钮的默认状态和交互细节：

麦克风按钮：默认开启（图标亮起），点击切换开关，同时静音/取消静音。

摄像头按钮：默认关闭（图标置灰），点击开启，画面自动出现在右下角画中画（现有功能，确保生效）。

停止共享按钮：点击后完全退出会议，返回待机界面，双方提示“会议已结束”。

步骤 3：隐藏所有技术配置项
3.1 设置面板清理

文件：SettingsModal.svelte 或对应设置组件

仅保留：皮肤切换、设备选择（麦克风/摄像头下拉框）、语言（如果有）。

移除或隐藏：信令端口、TURN 地址、局域网 IP 等所有网络配置字段。

如果这些字段不在当前 UI 中，则无需操作。

3.2 首次运行权限引导

检测是否首次运行（localStorage 标记），如果是，弹出极简权限提示：“本应用需要访问屏幕和网络，请点击允许”，点击“知道了”后存入标记，不再显示。

步骤 4：验证与测试
修改完成后，请使用以下方式验证：

双实例测试：运行 test-two.bat，一个窗口开始会议，另一个窗口输入短码加入，确认画面、音频正常。

局域网测试（如果有条件）：在另一台电脑上运行应用，输入主机的短码，确认连接成功。

跨网络测试：用手机热点连接两台电脑，重复上述测试，确认 TURN 中继工作。
