# 指令：添加摄像头开关按钮（默认关闭）

## 目标
在底部工具栏增加一个“摄像头”按钮，控制是否采集并发送本地摄像头画面。按钮默认关闭，点击后开启摄像头，画面以画中画形式叠加在共享屏幕右下角（若正在共享），或独立显示（若仅开启摄像头未共享）。

## 要求

### 1. 按钮 UI
- 位置：底部工具栏，与“麦克风”、“共享屏幕”等按钮并列。
- 图标：使用摄像头图标（可用 Font Awesome 的 `fa-video` 或 `fa-camera`），关闭状态显示斜杠或灰色。
- 样式跟随当前皮肤主题（变量化）。
- 初始状态：关闭（图标置灰，无摄像头流）。
- 点击切换：开 → 关，关 → 开。

### 2. 摄像头流管理
- 开启时调用 `navigator.mediaDevices.getUserMedia({ video: true, audio: false })` 获取摄像头流。
- 将摄像头视频轨道添加到已有的 `RTCPeerConnection`（如果存在且连接状态为 `connected`），使用 `pc.addTrack(cameraTrack, localStream)`。
- 若尚未建立 PeerConnection（比如未开始共享），则暂存摄像头流，待连接建立后再添加。
- 关闭时停止所有摄像头轨道（`track.stop()`），并从 PeerConnection 中移除对应的发送器（或直接关闭整个流后再重新协商）。

### 3. 画中画显示
- 当摄像头开启时，在共享屏幕视图（或主界面）右下角创建一个悬浮视频元素，播放本地摄像头流。
- 该视频元素可拖动、可调整大小，默认尺寸 160x120，保持 4:3 比例。
- 关闭摄像头时移除该视频元素。
- 若当前没有共享屏幕，则摄像头画面占据主区域（或居中显示）。

### 4. 状态同步与权限
- 摄像头开启/关闭状态通过现有信令通道通知对方（如通过 DataChannel 发送 JSON 消息），以便对方 UI 显示“对方已开启摄像头”。
- 处理权限拒绝：若无摄像头权限或设备不存在，弹出中文提示“无法访问摄像头，请检查权限设置”，按钮保持关闭。
- 支持切换摄像头设备（可选）：长按或右键按钮列出可用摄像头列表，选择切换。

### 5. 代码修改范围
- 新增 `src/renderer/src/CameraButton.svelte`（或直接在工具栏组件中集成）。
- 修改 WebRTC 连接管理模块（如 `src/lib/webrtc.js`），添加摄像头轨道管理方法 `addCameraTrack(stream)`、`removeCameraTrack()`。
- 修改主界面布局，在右下角嵌入画中画视频容器（可复用现有 `PictureInPicture.svelte` 或新建）。
- 确保与现有皮肤系统兼容，所有新增元素使用 CSS 变量。

### 6. 测试验证
- 启动第一个实例，开启屏幕共享，点击摄像头按钮，本地出现画中画，对方也能看到你的摄像头画面（在对方共享画面上叠加或独立显示）。
- 关闭摄像头按钮，本地画中画消失，对方也停止显示。
- 未共享屏幕时点击摄像头，本地显示摄像头画面（全屏或居中）。
- 权限拒绝时出现提示，按钮不开启。
